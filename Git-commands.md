1. Подготовка (Локально)
Если вы только создали папку с кодом:

git init — превращает обычную папку в Git-репозиторий (создает скрытую папку .git).

git status — показывает состояние файлов: что изменилось, что будет добавлено в коммит, а что Git «не видит».

2. Очистка и игнорирование (Ваш случай)
Если вы случайно добавили лишние файлы (bin, obj) и исправили .gitignore:

git rm -r --cached . — «забыть» все файлы в текущей папке. Сами файлы остаются на диске, но Git перестает их отслеживать.

git add . — добавить все файлы заново. На этом этапе Git заглянет в ваш исправленный .gitignore и пропустит ненужные папки.

git status — проверка: вы должны увидеть список файлов со статусом deleted (те, что ушли в игнор).

3. Сохранение и отправка
git commit -m "сообщение" — создает «снимок» (версию) вашего кода с кратким описанием изменений.

git branch -M main — переименовывает основную ветку в main (современный стандарт GitHub).

git remote add origin <ссылка> — связывает ваш компьютер с репозиторием на сервере GitHub.

git push -u origin main — отправляет ваши коммиты в облако. Флаг -u запоминает настройки, чтобы в следующий раз писать просто git push.

4. Работа в процессе
Когда вы написали новый код и хотите его отправить:

git add . (подготовить изменения).

git commit -m "решил задачу №5" (сохранить локально).

git push (отправить на GitHub).
