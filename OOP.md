Что такое класс и объект

Класс — это чертёж/шаблон для создания объектов.
Описывает данные (поля) и поведение (методы).

Объект — конкретный экземпляр класса, созданный через new.

Пример кода

public class Person { /* чертёж */ }

Person person1 = new Person();  // объект №1
Person person2 = new Person();  // объект №2 — отдельный от person1
──────────────────────────────────────────────────────────────────────
Конструкторы

Специальный метод, который вызывается при new.
Используется для инициализации полей.
Может быть несколько (перегрузка)

Пример кода
public class Person
{
    public string Name;
    public int Age;

    public Person()                          // по умолчанию
    {
        Name = "Неизвестно";
        Age = 0;
    }

    public Person(string name, int age)       // с параметрами
    {
        Name = name;
        Age = age;
    }
}

// Использование
Person p1 = new Person();                        // вызов по умолчанию
Person p2 = new Person("Анна", 25);               // вызов с параметрами
──────────────────────────────────────────────────────────────────────
Поля vs Свойства

Поля — хранилище данных, должны быть private.

Свойства — контролируемый доступ к полям + возможность добавить логику (валидацию).

Пример кода
public class Person
{
    private int _age;                          // поле — скрыто

    public int Age                             // свойство с валидацией
    {
        get => _age;
        set
        {
            if (value < 0)
                throw new ArgumentException("Возраст не может быть отрицательным");
            _age = value;
        }
    }

    // Автосвойство (когда валидация пока не нужна)
    public string Name { get; set; }
}

Инкапсуляция

Принцип: скрывать внутреннюю реализацию, показывать только необходимый интерфейс.

Разница между Абстракцией и инкапсуляцией
«Абстракция — это ЧТО объект делает (внешний интерфейс), а инкапсуляция — это КАК он это делает внутри и как он себя при этом защищает».

Реализация: private поля + public свойства.

Зачем:
• Защита данных от некорректных значений
• Гибкость изменений внутри класса без ломки внешнего кода
• Объект всегда в валидном состоянии

Пример кода
private string _name;

public string Name
{
    get => _name;
    set => _name = string.IsNullOrWhiteSpace(value) ? "Неизвестно" : value.Trim();
}

// Внешний код не может сломать объект:
person.Name = null;      // станет "Неизвестно"
person.Age = -5;          // бросит исключение
──────────────────────────────────────────────────────────────────────
Ключевое слово this

Ссылается на текущий объект внутри класса.
Полезно, чтобы отличать поля от параметров с одинаковыми именами.

Пример кода
public Person(string name, int age)
{
    this.Name = name;   // this.Name — свойство, name — параметр
    this.Age = age;
}

Почему это важно прямо сейчас
Раньше мы работали только с готовыми типами (int, string, List<T>).
Теперь мы сами создаём свои типы данных — это фундамент ООП и основа всех реальных приложений.
